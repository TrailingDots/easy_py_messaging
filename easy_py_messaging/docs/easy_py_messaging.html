Easy Py Messaging

Features of Easy Py Messaging

Easy Py Messaging provides a small cross-platform python coded toolkit for easy, reliable communications between nodes, within a single desktop for multiple processes or just within a single process. Just send message and the API delivers your message.

Easy Py Messaging provides a wrapper around ZeroMQ to implement a logger, a directory naming service and an API for user implementions. Multiple additional utilites support the code.

Feature Overview

  • Python API using the industry standard ZeroMQ library. ZeroMQ has a proven track record for speed of delivery and a robust API.
  • Asynchronous handling. Messages queues provide the ability to fire-and-forget. For input, a queue query may either await a message or contnue.
  • Robust messaging. ZeroMQ permits various processes to go down while supplying recovery connections when the processes restarts. Sorry, persistence is not currently implemented.
  • Symmetrical connections. Messages use TCP for bi-directional connections.
  • Multiple support utilties. These utilities support your development and debugging efforts.
    • logCollector. The primary tool for collecting and storing logs.
    • logCmd. Send logs and commands from the shell command line.
    • loggingLoopApp. Send a log message once a second for debugging configurations.
    • logFilterApp. Given a log file, extract relevant information.
    • dirSvc. A directory naming service. Connect your applications by name instead of port. This greatly simplifies your configuation files.
    • loggingSpeedTest. How fast does this system operate with your network configuration?

The Tools

logCollector - Collect the logs

The logCollector documentation provides a thorough overview of log collection. A decent logging system significantly eases development because it allows simple tracking of information between various nodes and processes.

logFilterApp - analyze logs

Once logs get written to a file, what happens? Log files can easily get large and manually skimming them becomes boring and prone to overlooking relevant information. logFilter provides elementary processing and filtering of these log files.

LoggingLoopApp - loop testing configurations

The loggingLoopApp gets used while developing applications. It sends logs to the logger once per second and provides feedback that the system for logs has a proper configuration.

loggingSpeedTest - how fast can you communicate?

How fast is your system? While speeds depend primarily upon your network, this utility will send 100,000 logs and report the time. What is yout timing for internal only, connected with ethernet cables, wireless connection?

dirSvc - Directory for Naming Servies

Easy Naming Service eases the configuration problems by supplying a symbolic naming connection between various services in your systems.

The Easy Messaging Utilties

This package contains multiple utilities to enhance the simple environment of the basic package.

logCmd - Command line messaging

The basic code of logCmd displays the ease of sending messages from an app. The additional command line options turn this code into a powerful command line utility. Interested developers could use this as the basis for a chat app.

dirSvc - The directory naming service

A directory naming service allows much more flexible configurations by using names for the various components of your system instead of obscure items such as IP addresses.

dirClient - Interrogating the nameing service.

This utility exercises the naming service. Useful for checking out configurations in development.

listening - Determing what process listens to a port

In the process of developing and debugging your systems, processes frequently continue to run while listening to a port. Unless a utility such as listening provides a mapping between the port number and the process, interpreting the output error message can prove tricky. This utility answers the frequent question on various forums "How do I find out which process is listening to a port?"
